数据库格式

COLMAP 将所有提取到的信息存储在单个 SQLite 数据库文件中。可以通过 COLMAP GUI 中的数据库管理工具包、提供的 C++ 数据库 API（参见 src/colmap/scene/database.h），或使用 Python 配合 pycolmap 来访问该数据库。

该数据库包含以下数据表：

  • rigs

  • cameras

  • frames

  • images

  • keypoints

  • descriptors

  • matches

  • two_view_geometries

要使用所需的模式初始化一个空的 SQLite 数据库文件，你可以在 GUI 中创建一个新项目，或运行 colmap database_creator 命令。

相机装置与传感器

相机装置与传感器（相机等）之间的关系是 1 对 N，其中一个传感器被选为参考传感器，以定义相机装置的原点。每个传感器只能属于一个相机装置。

相机装置与帧

相机装置与帧之间的关系是 1 对 N，其中帧定义了相机装置的一个具体实例，其中所有或部分传感器在同一时刻进行曝光采集。

相机与图像

相机与图像之间的关系是 1 对 N。这对运动恢复结构（SfM）有重要意义，因为一个相机共享相同的内参（焦距、主点、畸变等），而每幅图像都有各自独立的外参（朝向和位置）。

相机的内参以连续的 float64 二进制 blob 形式存储，其顺序按 src/colmap/sensor/models.h 中所指定的方式排列。COLMAP 只使用被图像引用的相机，其他所有相机都会被忽略。

images 表中的 name 列是图像文件夹中唯一的相对路径。因此，只要相对文件夹结构得以保留，数据库文件和图像文件夹就可以移动到不同的位置。

在手动向数据库中插入图像和相机时，请确保所有标识符均为正且非零，即 image_id > 0 且 camera_id > 0。

关键点与描述子

检测到的关键点以行优先的 float32 二进制 blob 形式存储，其中前两列分别是关键点在图像中的 X 和 Y 位置。COLMAP 采用的约定是：图像左上角的坐标为 (0, 0)，最左上角像素的中心坐标为 (0.5, 0.5)。如果关键点有 4 列，则特征几何为相似变换，第三列是特征的尺度，第四列是特征的朝向（遵循 SIFT 约定）。如果关键点有 6 列，则特征几何为仿射变换，最后 4 列编码其仿射形状（详见 src/colmap/feature/types.h）。

提取到的描述子以行优先的二进制 blob 形式存储，其中每一行描述了 keypoints 表中对应条目的特征外观。数据类型和维度取决于特征提取器：

  • SIFT：uint8 描述子，128 维（每个特征 128 字节）。

  • ALIKED：float32 描述子，128 维（每个特征 512 字节）。

descriptors 表中的 cols 列指定每行描述子的字节数。对于 uint8 描述子，该值等于描述子维度。对于 float32 描述子，该值等于 4 * dimension。

在这两个表中，rows 表指定每幅图像检测到的特征数量，而 rows=0 表示某幅图像没有特征。对于特征匹配和几何验证，每幅图像都必须有对应的 keypoints 和 descriptors 条目。请注意，只有配合快速空间验证的词汇树匹配才需要局部特征几何的有效值，即只需提供 X 和 Y，其他关键点列可以设为零。重建流程的其余部分仅使用关键点的位置。

匹配与双视图几何

特征匹配将其输出存储在 matches 表中，几何验证则存储在 two_view_geometries 表中。COLMAP 仅使用 two_view_geometries 中的数据进行重建。这两个表中的每个条目都存储了两幅唯一图像之间的特征匹配，其中 pair_id 是上三角匹配矩阵中行优先的线性索引，按如下方式生成:

def image_ids_to_pair_id(image_id1, image_id2):
    if image_id1 > image_id2:
        return 2147483647 * image_id2 + image_id1
    else:
        return 2147483647 * image_id1 + image_id2

并且图像标识符可以从 pair_id 唯一确定，方式如下:

def pair_id_to_image_ids(pair_id):
    image_id2 = pair_id % 2147483647
    image_id1 = (pair_id - image_id2) / 2147483647
    return image_id1, image_id2

pair_id 使得高效的数据库查询成为可能，因为 matches 表可能包含数亿条条目。该方案将数据库中图像的最大数量限制为 2147483647（有符号 32 位整数的最大值），即 image_id 必须小于 2147483647。

matches 表中的二进制 blob 是行优先的 uint32 矩阵，其中左列是指向 image_id1 特征的从零开始的索引，第二列是指向 image_id2 特征的索引。cols 列必须为 2，rows 列指定特征匹配的数量。

two_view_geometries 表中的 F、E、H blob 以行优先的 float64 格式存储为 3x3 矩阵。config 值的含义记录在 src/colmap/estimators/two_view_geometry.h 源文件中。
