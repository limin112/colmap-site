.. _features:

特征提取与匹配
==============

COLMAP 支持多种特征提取与特征匹配算法。本页介绍如何通过命令行界面或图形用户界面在它们
之间进行切换。


特征提取器类型
--------------

可用的特征提取器类型如下：

- ``SIFT``：尺度不变特征变换（默认）。经典且经过最广泛测试的特征提取器。产生 128 维的
  uint8 描述子。

- ``ALIKED``：A Lighter Keypoint and Descriptor Extractor（更轻量的关键点与描述子提取器）。
  一种基于学习的特征提取器，产生浮点型描述子。要求在编译时启用 ONNX 支持
  （``-DONNX_ENABLED=ON``）。

通过命令行选择特征提取器类型::

    $ colmap feature_extractor \
        --database_path $DATASET_PATH/database.db \
        --image_path $DATASET_PATH/images \
        --FeatureExtraction.type ALIKED_N16ROT \
        --AlikedExtraction.max_num_features 2048

对于 SIFT（默认），你可以省略类型，也可以显式设置它::

    $ colmap feature_extractor \
        --database_path $DATASET_PATH/database.db \
        --image_path $DATASET_PATH/images \
        --FeatureExtraction.type SIFT \
        --SiftExtraction.max_num_features 8192

在 GUI 中，打开 ``Processing > Feature extraction`` 并在点击 Extract 之前选择所需的
标签页（SIFT、ALIKED 等）。


特征匹配器类型
--------------

可用的特征匹配器类型如下：

- ``SIFT_BRUTEFORCE``：针对 SIFT 描述子优化的暴力匹配（默认）。使用带比值检验的 L2 距离。

- ``ALIKED_BRUTEFORCE``：针对 ALIKED 描述子的暴力匹配。使用余弦相似度。要求在编译时启用
  ONNX 支持。

- ``SIFT_LIGHTGLUE``：针对 SIFT 描述子，使用 LightGlue 模型的基于神经网络的匹配。相比
  暴力匹配，它通常能产生更多的匹配和更高的内点率，尤其是对于视角或光照变化较大的高难度
  图像对。要求在编译时启用 ONNX 支持。

- ``ALIKED_LIGHTGLUE``：针对 ALIKED 描述子，使用 LightGlue 模型的基于神经网络的匹配。
  要求在编译时启用 ONNX 支持。

通过命令行选择特征匹配器类型::

    $ colmap exhaustive_matcher \
        --database_path $DATASET_PATH/database.db \
        --FeatureMatching.type ALIKED_BRUTEFORCE \
        --AlikedMatching.min_cossim 0.85

对于 SIFT 匹配（默认）::

    $ colmap exhaustive_matcher \
        --database_path $DATASET_PATH/database.db \
        --FeatureMatching.type SIFT_BRUTEFORCE \
        --SiftMatching.max_ratio 0.8

在 GUI 中，打开 ``Processing > Feature matching``，选择任一匹配标签页（Exhaustive、
Sequential 等），并在共享选项区域的 “Type” 下拉框中选择匹配器类型。


兼容的提取器与匹配器类型
------------------------

特征提取器与匹配器的类型应当相互兼容：

- 将 ``SIFT`` 提取与 ``SIFT_BRUTEFORCE`` 或 ``SIFT_LIGHTGLUE`` 匹配配合使用。
- 将 ``ALIKED_*`` 提取与 ``ALIKED_BRUTEFORCE`` 或 ``ALIKED_LIGHTGLUE`` 匹配配合使用。

混用不兼容的类型（例如，SIFT 特征配合 ALIKED 匹配器）将导致运行时错误。不要在同一个
数据库中混用不同的特征提取器类型（例如，SIFT 和 ALIKED）。


ALIKED 模型变体
---------------

ALIKED 需要一个 ONNX 模型文件。提供了若干种在速度与精度之间有不同取舍的模型变体：

- ``aliked-n16rot``：更快，并针对一定程度的视角不变性进行了训练。128 维描述子。
- ``aliked-n32``：开销更大，但未专门针对视角不变性进行训练，128 维描述子。

使用 ``--AlikedExtraction.*_model_path`` 指定模型路径。如果该路径是一个 URL，COLMAP 将
自动下载并缓存该模型。你可以从发布页面 https://github.com/colmap/colmap/releases/
下载不同的 ALIKED 模型。
