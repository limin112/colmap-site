COLMAP 4.0.0 (03/14/2026)

新功能（New Features）

  • 将 GLOMAP 全局 SfM 流程作为增量/分层 mapper 之外的一等替代方案集成进 COLMAP，可通过 global_mapper 和 automatic_reconstructor --mapper GLOBAL 命令使用。作为此次迁移的一部分，对 GLOMAP 代码库应用了许多修复和改进。 GLOMAP 今后将通过 COLMAP 仓库进行维护。 该全局流程使用视图图标定从双视图几何中估计内参，这可能产生与增量流程的 自标定不同的相机参数。（Integrated GLOMAP global SfM pipeline into COLMAP as a first-class alternative to the incremental/hierarchical mappers, available via the global_mapper and automatic_reconstructor --mapper GLOBAL commands. Many fixes and improvements have been applied to the GLOMAP codebase as part of this migration. GLOMAP is maintained through the COLMAP repository going forward. The global pipeline uses view graph calibration to estimate intrinsics from two-view geometries, which may produce different camera parameters compared to the incremental pipeline's self-calibration.）

  • 通过 ONNX 支持添加了 ALIKED（N16Rot/N32）特征提取。支持暴力匹配 和 LightGlue 匹配，以及使用预训练词汇树的可扩展匹配。（Added ALIKED (N16Rot/N32) feature extraction through ONNX support. Support for brute-force and LightGlue matching as well as scalable matching with pre-trained vocabulary trees.）

  • 为 SIFT 和 ALIKED 添加了 LightGlue ONNX 特征匹配。（Added LightGlue ONNX feature matching for SIFT and ALIKED.）

  • 为特征提取器和匹配器添加了 Python 绑定。（Added Python bindings for feature extractor and matcher.）

  • 添加了从 EXIF 读取图像方向并在特征提取/特征匹配期间自动旋转图像的支持， 以更好地应对旋转视角变化。（Added support for reading image orientation from EXIF and auto-rotating images during feature extraction/matching for better robustness against rotational viewpoint changes.）

  • 添加了使用广义相对位姿估计器的无结构图像注册回退方案， 用于注册没有三视图重叠的图像。（Added structure-less image registration fallback using generalized relative pose estimator for registering images without 3-view overlap.）

  • 添加了除法相机模型（SIMPLE_DIVISION、DIVISION）。（Added division camera models (SIMPLE_DIVISION, DIVISION).）

  • 添加了不带畸变参数的鱼眼相机模型（FISHEYE）。（Added fisheye camera model without distortion parameters (FISHEYE).）

  • 通过单一位姿参数块将 Ceres 光束法平差性能提升了约 10%。（Improved Ceres bundle adjustment performance by ~10% through single pose parameter block.）

  • 通过解析雅可比矩阵，将 SIMPLE_RADIAL 相机模型和平凡帧的 Ceres 光束法平差性能 提升了约 15%。（Improved Ceres bundle adjustment performance by ~15% for SIMPLE_RADIAL camera model and trivial frames through analytical Jacobians.）

  • 添加了抽象的通用光束法平差器和求解器摘要，为支持不同的优化后端做准备。（Added abstract general bundle adjuster and solver summary in preparation for supporting different optimization backends.）

  • 用 OpenImageIO 替换 FreeImage，使图像 I/O 速度提升约 2.5 倍、支持更多 图像格式，并使用具有安全修复等的积极维护依赖。（Replaced FreeImage with OpenImageIO for ~2.5x faster image I/O, support for more image formats, and using an actively maintained dependency with security fixes, etc.）

  • 添加了使用已知重建的引导式几何验证。（Added guided geometric verification using a known reconstruction.）

  • 将视图图标定添加为新模块。（Added view graph calibration as a new module.）

  • 添加了模型聚类，可基于场景连通性将大型重建划分为 更小、更易管理的子模型。（Added model clustering to partition large reconstructions into smaller, manageable sub-models based on scene connectivity.）

  • 添加了使用二次误差度量（QEM）抽取的网格简化。（Added mesh simplification using Quadric Error Metric (QEM) decimation.）

  • 添加了网格纹理映射，可从已标定图像生成带纹理的网格。（Added mesh texture mapping for producing textured meshes from calibrated images.）

  • 添加了在图像去畸变期间指定 JPEG 质量的选项。（Added option to specify JPEG quality during image undistortion.）

  • 添加了加载带 alpha 通道位图的支持。（Added support for loading bitmaps with alpha channel.）

  • 添加了在光束法平差器中控制三维点优化的选项。（Added option to control refinement of 3D points in bundle adjuster.）

  • 添加了在绝对位姿优化中对位置先验的可选支持。（Added optional support for position prior in absolute pose refinement.）

  • 添加了 supernodal CHOLMOD L1 求解器支持。（Added supernodal CHOLMOD L1 solver support.）

  • 为 (LO)RANSAC 循环添加了多线程支持。（Added multi-threading support for (LO)RANSAC loop.）

  • 添加了通过 SSL_CERT_FILE 和 SSL_CERT_DIR 环境变量支持自定义 SSL 证书位置。（Added support for custom SSL certificate locations through SSL_CERT_FILE and SSL_CERT_DIR environment variables.）

  • 添加了对带 CUDA 的静态 Windows 构建的支持。（Added support for static Windows builds with CUDA.）

  • 公开了 pycolmap.match_from_pairs，用于在 GPU 上进行自定义配对匹配。（Exposed pycolmap.match_from_pairs for custom pair matching on GPU.）

  • 为深度图和法向图添加了 Python 绑定。（Added Python bindings for depth and normal maps.）

  • 为所有代码添加了 Python 类型检查（mypy）。（Added Python type checking (mypy) for all code.）

  • 改进了增量 mapper 和三角化器的 Python 绑定。（Improved incremental mapper and triangulator Python bindings.）

  • 添加了在 GUI 中可视化（带纹理的）网格的支持。（Added support for visualizing (textured) meshes in the GUI.）

  • 在 GUI 中添加了用于模型导入的拖放文件夹支持。（Added drag-and-drop folder support for model import in the GUI.）

  • 在 GUI 中添加了 PLY 导入的拖放支持。（Added drag-and-drop support for PLY import in the GUI.）

  • 在 macOS 上的 GUI 中切换为原生菜单栏。（Switched to native menu bar in GUI on macOS.）

  • 添加了构建共享库的支持。（Added support for building shared libraries.）

  • 通过并行读取输入改进了 patch match 立体的性能。（Improved patch match stereo performance by reading inputs in parallel.）

  • 减少了 GUI 中模型查看器的内存占用。（Reduced model viewer memory usage in the GUI.）

  • 添加了在 GPU 求解器失败时自动回退到 BA CPU 求解器。（Added automatic fallback to BA CPU solver if GPU solver fails.）

  • 其他小改进请参见下方的完整变更列表。（For other minor improvements, see full list of changes below.）

重大变更（Breaking Changes）

  • 用 OpenImageIO 替换 FreeImage 进行图像 I/O。在支持的图像格式和加载的像素值 方面可能存在细微差异。（Replaced FreeImage with OpenImageIO for image I/O. There may be slight differences in terms of supported image formats and loaded pixel values.）

  • 由于已到生命周期终点，放弃了对 Python 3.9 的支持。（Dropped support for Python 3.9 due to EOL.）

  • TwoViewGeometry 的字段 cam2_from_cam1、E、F、H 现在为 std::optional。（TwoViewGeometry fields cam2_from_cam1, E, F, H are now std::optional.）

  • Rigid3d 和 Sim3d 参数存储为单一向量。 请使用 rotation() 和 translation() 访问器方法，而不是直接访问成员。（Rigid3d and Sim3d parameters stored as single vector. Use rotation() and translation() accessor methods instead of direct member access.）

  • 光束法平差代价函子使用单一位姿参数块。（Bundle adjustment cost functors use single pose parameter blocks.）

  • BundleAdjuster 现在返回 std::shared_ptr<BundleAdjusterSummary> 而不是 ceres::SolverSummary。（BundleAdjuster now returns std::shared_ptr<BundleAdjusterSummary> instead of ceres::SolverSummary.）

  • Ceres 相关的 CLI 选项以 BundleAdjustmentCeres.* 为前缀。（Ceres-related CLI options are prefixed with BundleAdjustmentCeres.*.）

  • 清理了 GPS/ENU 坐标转换功能，破坏了 Python 接口。（GPS/ENU coordinate conversion functionality cleaned up, breaking the Python interface.）

  • 特征描述子现在与一个类型相关联，该类型会贯穿整个流程和存储进行传播。 如果描述子类型不一致，则匹配失败。（Feature descriptors are now associated with a type that is propagated through the pipeline and storage. Fails matching if descriptor types are inconsistent.）

  • 位姿先验与通用传感器测量数据相关联。（Pose priors associated with generic sensor measurement data.）

  • DatabaseCache 的初始化整合为单一选项结构体， 而不是分散的参数。（DatabaseCache initialization consolidated into a single options struct instead of scattered arguments.）

  • 在整个 C++ 代码库中将路径从 std::string 切换为 std::filesystem::path。 Python 的 pathlib.Path 对象现在会自动转换。 对 Python 接口没有破坏性影响。（Switched paths from std::string to std::filesystem::path throughout the C++ codebase. Python pathlib.Path objects are now automatically converted. No breaking impact on the Python interface.）

重大变更（Breaking Changes）

  • 重建和数据库的序列化包含一个新的抽象：相机装置（rigs）和帧（frames）。重建输出包含两个新文件 rigs.{bin,txt} 和 frames.{bin,txt}。数据库包含新表：rigs、rig_sensors、frames、frames_data。从现有的重建和数据库（不含 rigs/frames）读取是完全向后兼容的，反之，使用旧代码读取新的重建（含 rigs/frames）是完全向前兼容的。（Serialization of reconstruction and database contains a new abstraction: rigs and frames.   • 传感器位姿（从而图像位姿）现在按如下方式组合：sensor_from_world = sensor_from_rig * rig_from_world。此前，image.cam_from_world 返回对位姿参数的引用。现在它返回位姿组合的一个副本：image.cam_from_world() = image.frame.rig.sensor_from_rig(image.camera.sensor_id) * image.frame.rig_from_world，其底层位姿参数存储在 rig 和 frame 对象中。（Sensor poses (and thus image poses) are now composed as: sensor_from_world = sensor_from_rig * rig_from_world. Previously, image.cam_from_world returned a reference to the pose parameters. Now it returns a copy of the pose composition: image.cam_from_world() = image.frame.rig.sensor_from_rig(image.camera.sensor_id) * image.frame.rig_from_world with the underlying pose parameters stored in the rig and frame objects.）

  • 默认的光束法平差器支持传感器装置，因此装置光束法平差器已被弃用。（Default bundle adjuster supports sensor rigs and thus rig bundle adjuster is deprecated.）

  • 基于 FLANN 的词汇树与 faiss 不兼容。如果未提供 vocab_tree_path，则会自动下载新的树，否则需要手动下载并更新。（FLANN-based vocabulary trees are incompatible with faiss. New trees automatically downloaded, if no vocab_tree_path is provided, otherwise manual download and update required.）
