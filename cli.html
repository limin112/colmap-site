命令行界面

命令行界面提供了对 COLMAP 全部功能的访问，可用于自动化脚本。每个核心功能都实现为 colmap 可执行文件的一个命令。运行 colmap -h 可列出所有可用命令（在 Windows 下为 COLMAP.bat -h）。注意，如果你从 CMake 构建文件夹运行 COLMAP，可执行文件位于 ./src/colmap/exe/colmap。要启动图形用户界面，请运行 colmap gui。

示例

假设你将项目的图像按如下结构存放:

/path/to/project/...
+── images
│   +── image1.jpg
│   +── image2.jpg
│   +── ...
│   +── imageN.jpg

则自动重建工具的命令为:

# The project folder must contain a folder "images" with all the images.
$ DATASET_PATH=/path/to/project

$ colmap automatic_reconstructor \
    --workspace_path $DATASET_PATH \
    --image_path $DATASET_PATH/images

注意，任何命令都可以使用 -h,--help 命令行参数列出其所有可用选项。如果你需要对重建 过程中各个参数进行更精细的控制，可以执行下面的命令序列，作为自动重建命令的替代方案:

# The project folder must contain a folder "images" with all the images.
$ DATASET_PATH=/path/to/dataset

$ colmap feature_extractor \
   --database_path $DATASET_PATH/database.db \
   --image_path $DATASET_PATH/images

$ colmap exhaustive_matcher \
   --database_path $DATASET_PATH/database.db

$ mkdir -p $DATASET_PATH/sparse

$ colmap mapper \
    --database_path $DATASET_PATH/database.db \
    --image_path $DATASET_PATH/images \
    --output_path $DATASET_PATH/sparse

$ mkdir -p $DATASET_PATH/dense

$ colmap image_undistorter \
    --image_path $DATASET_PATH/images \
    --input_path $DATASET_PATH/sparse/0 \
    --output_path $DATASET_PATH/dense \
    --output_type COLMAP \
    --max_image_size 2000

$ colmap patch_match_stereo \
    --workspace_path $DATASET_PATH/dense \
    --workspace_format COLMAP \
    --PatchMatchStereo.geom_consistency true

$ colmap stereo_fusion \
    --workspace_path $DATASET_PATH/dense \
    --workspace_format COLMAP \
    --input_type geometric \
    --output_path $DATASET_PATH/dense/fused.ply

$ colmap poisson_mesher \
    --input_path $DATASET_PATH/dense/fused.ply \
    --output_path $DATASET_PATH/dense/meshed-poisson.ply

$ colmap delaunay_mesher \
    --input_path $DATASET_PATH/dense \
    --output_path $DATASET_PATH/dense/meshed-delaunay.ply

# Optionally simplify a dense mesh to reduce its size.
$ colmap mesh_simplifier \
    --input_path $DATASET_PATH/dense/meshed-poisson.ply \
    --output_path $DATASET_PATH/dense/meshed-poisson-simplified.ply \
    --MeshSimplification.target_face_ratio 0.25

# Optionally texture a mesh using the undistorted images.
$ colmap mesh_texturer \
    --workspace_path $DATASET_PATH/dense \
    --input_path $DATASET_PATH/dense/meshed-poisson.ply \
    --output_path $DATASET_PATH/dense/textured

如果想使用全局 SfM 流程而非增量式建图器，请将 mapper 步骤替换为 global_mapper。 全局建图器依赖良好的焦距先验，因此如果没有可靠的内参可用（例如来自 EXIF 或实验室标定）， 你应当先运行 view_graph_calibrator。该步骤是可选的，但建议执行以提升全局 SfM 的质量， 这也一直是 GLOMAP 中的默认做法。注意， view_graph_calibrator 会就地修改数据库中的相机内参和两视图几何，因此建议在数据库的 副本上进行操作:

$ colmap feature_extractor \
   --database_path $DATASET_PATH/database.db \
   --image_path $DATASET_PATH/images

$ colmap exhaustive_matcher \
   --database_path $DATASET_PATH/database.db

# Optional but often needed: calibrate intrinsics from the view graph.
# This modifies the database in-place, so work on a copy.
$ cp $DATASET_PATH/database.db $DATASET_PATH/database_global.db
$ colmap view_graph_calibrator \
    --database_path $DATASET_PATH/database_global.db

$ mkdir -p $DATASET_PATH/sparse

$ colmap global_mapper \
    --database_path $DATASET_PATH/database_global.db \
    --image_path $DATASET_PATH/images \
    --output_path $DATASET_PATH/sparse

如果你想在没有连接显示器的计算机上运行 COLMAP（例如集群或云服务），只要系统支持，COLMAP 会自动切换为使用 CUDA。如果没有可用的支持 CUDA 的设备，你可以通过设置 --FeatureExtraction.use_gpu 0 和 --FeatureMatching.use_gpu 0 选项，手动选择 使用基于 CPU 的特征提取与匹配。

命令

下面的列表简要说明了每个命令的功能，这些命令均可通过 colmap [command] 调用：

  • gui：图形用户界面，更多信息参见 图形用户界面。

  • automatic_reconstructor：为一组输入图像自动重建稀疏模型和稠密模型。主要选项包括 --quality（LOW、MEDIUM、HIGH、EXTREME）、--data_type（INDIVIDUAL、VIDEO、 INTERNET，用于针对不同拍摄场景调整设置）、--feature（SIFT、ALIKED，用于选择特征 提取算法）、--mapper（INCREMENTAL、HIERARCHICAL、GLOBAL，用于选择 SfM 流程）以及 --mesher（POISSON、DELAUNAY，用于选择表面重建方法）。

  • project_generator：以不同的质量设置生成项目文件。

  • feature_extractor、feature_importer：对一组图像执行特征提取或导入特征。

  • exhaustive_matcher、vocab_tree_matcher、sequential_matcher、 spatial_matcher、transitive_matcher、matches_importer： 在执行特征提取之后执行特征匹配。

  • geometric_verifier：对数据库中已有的特征匹配运行独立的几何验证。该步骤会为已匹配的 图像对估计两视图几何（基础矩阵/本质矩阵、单应矩阵）。

  • guided_geometric_verifier：在已有稀疏重建的引导下运行几何验证。利用已知的相对相机 位姿来改进匹配验证结果。

  • mapper：在执行特征提取与匹配之后，使用 SfM 对数据集进行稀疏三维重建/建图。

  • global_mapper：使用全局 SfM 流程进行稀疏三维重建。与增量式的 mapper 不同， 全局方法使用旋转平均和全局定位同时求解所有相机位姿。对于大型数据集，这种方法可能更快， 但对外点的鲁棒性可能较差。 全局建图器依赖于足够好的焦距先验才能表现良好。请在 global_mapper 之前运行 view_graph_calibrator 以标定相机内参并从视图图估计相对位姿，或者手动提供相机标定。

  • pose_prior_mapper：使用位姿先验进行稀疏三维重建/建图。

  • hierarchical_mapper：在执行特征提取与匹配之后，使用层次化 SfM 对数据集进行稀疏 三维重建/建图。该方法通过将场景划分为相互重叠的子模型，然后独立地重建每个子模型，从而 并行化重建过程。最后，将这些相互重叠的子模型合并为单个重建。 建议在此步骤之后运行若干轮点三角化和光束法平差。

  • image_undistorter：对图像进行去畸变，和/或将其导出以用于 MVS 或外部稠密重建软件 （例如 CMVS/PMVS）。

  • image_rectifier：对相机进行立体校正并对图像去畸变，以用于立体视差估计。

  • image_filterer：从稀疏重建中过滤图像。

  • image_deleter：从稀疏重建中删除特定图像。

  • patch_match_stereo：在运行 image_undistorter 初始化工作空间之后，使用 MVS 进行稠密三维重建/建图。

  • stereo_fusion：将 patch_match_stereo 的结果融合为带颜色的点云。

  • poisson_mesher：使用泊松表面重建对融合后的点云进行网格化。

  • delaunay_mesher：使用 Delaunay 三角化上的图割和可见性投票，对重建得到的稀疏或 稠密点云进行网格化。

  • mesh_simplifier：使用二次误差度量（QEM）抽取法简化三角网格（PLY 格式）。该步骤 在保持网格整体形状和外观的同时减少其面数。主要选项包括 --MeshSimplification.target_face_ratio，用于控制要保留的面的比例（默认 0.1）、 --MeshSimplification.max_error，用于设置最大二次误差阈值（0 = 禁用），以及 --MeshSimplification.boundary_weight，用于控制边界边的保留程度（默认 1000）。 支持通过 --MeshSimplification.num_threads 进行多线程初始化。

  • mesh_texturer：使用已标定的多视图图像，为三角网格生成纹理图集和 UV 坐标。

  • image_registrator：将数据库中的新图像与已有模型进行配准，例如在运行 mapper 之后，对数据库中新添加的图像提取特征并匹配时使用。注意，此过程不执行光束法平差或三角化。

  • point_triangulator：使用数据库中的特征匹配，对已有模型中已配准图像的所有观测进行 三角化。

  • point_filtering：通过强制执行某些准则（例如最小轨迹长度、最大重投影误差等）来过滤 模型中的稀疏点。

  • bundle_adjuster：对重建好的场景运行全局光束法平差，例如当需要优化内参时，或在运行 image_registrator 之后使用。

  • database_cleaner：清理特定的或全部数据库表。

  • database_creator：创建一个空的 COLMAP SQLite 数据库，并写入必要的数据库模式信息。

  • database_merger：将两个数据库合并为一个新的数据库。注意，相机不会被合并，且唯一的 相机和图像标识符在合并过程中可能发生变化。

  • model_analyzer：打印关于重建的统计信息。

  • model_clusterer：将一个重建拆分为更小的子模型簇。对于管理和处理大规模重建很有用。

  • model_aligner：将模型对齐/地理配准到由给定相机中心确定的坐标系。

  • model_orientation_aligner：使用曼哈顿世界假设对齐模型的坐标轴。

  • model_comparer：比较两个重建的统计信息。

  • model_converter：将 COLMAP 导出格式转换为另一种格式，例如 PLY 或 NVM。

  • model_cropper：将模型裁剪到以 GPS 或模型坐标系描述的特定包围盒。

  • model_merger：如果两个互不连通的重建具有共同的已配准图像，则尝试将它们合并。

  • model_splitter：根据文件（包含包围盒坐标、子模型最大范围，或每个维度上的细分数量） 将模型划分为矩形子模型。

  • model_transformer：变换模型的坐标系。

  • color_extractor：为模型的所有三维点提取平均颜色。

  • rig_configurator：在特征提取之后配置相机装置（rig）和帧。

  • vocab_tree_builder：根据一个已提取图像的数据库创建词汇树。这是一个离线过程，只需运行 一次，同一棵词汇树即可重用于其他数据集。注意，作为经验法则，你使用的特征数量应至少是视觉 单词数量的 10-100 倍。 预训练的树可从 https://demuc.de/colmap/ 下载。 如果你想构建一棵在精度/召回率与速度之间具有不同权衡的自定义树，这会很有用。

  • vocab_tree_retriever：执行基于词汇树的图像检索。

  • rotation_averager：在视图图上运行独立的旋转平均。从两两相对旋转估计全局相机旋转。

  • view_graph_calibrator：使用视图图标定相机内参。从两两几何关系估计焦距和其他内参。 如果没有已知的良好相机内参先验，应在 global_mapper 之前运行此命令，因为全局建图器 依赖于足够好的焦距先验才能表现良好。

可视化

如果你想快速可视化稀疏或稠密重建流程的输出，COLMAP 为你提供了以下几种可能：

  • 通过 mapper 获得的稀疏点云可以在 COLMAP GUI 中通过导入模型文件来可视化：选择 File > Import Model，然后选择包含稀疏模型文件（cameras.txt、 images.txt、points3D.txt 等）的文件夹。

  • 通过 stereo_fusion 获得的稠密点云可以在 COLMAP GUI 中通过导入 fused.ply 来可视化：选择 File > Import Model from...，然后选择文件 fused.ply。

  • 通过 poisson_mesher 或 delaunay_mesher 获得的稠密网格模型 meshed-*.ply 目前无法用 COLMAP 可视化，你可以改用外部查看器，例如 Meshlab。使用 mesh_simplifier 命令可以缩小网格规模，以加快可视化或下游处理。使用 mesh_texturer 命令可以生成带有 纹理图集的带纹理网格，该网格可以在 Meshlab 或其他三维查看器中可视化。
