01

它到底是什么？What is COLMAP

COLMAP 回答一个看似简单的问题：给我一组照片，相机当时站在哪、世界长什么样？ 它从二维像素里反推出三维几何，整个过程拆成两幕——

第一幕 SfM · 运动恢复结构

从照片反推出每张照片的拍摄位置与朝向（相机位姿），同时得到一副稀疏的三维「骨架」点云。先搞清楚相机在哪。

第二幕 MVS · 多视图立体

在已知相机位姿的前提下，把稀疏骨架填充成稠密点云，再重建出带纹理的网格。把世界填满。

一句话记住它：稀疏先行，稠密在后。MVS 完全建立在 SfM 算出的相机位姿之上——这是理解整条流程的总开关。

03

7 个关键概念Key concepts you must know

◆ 视觉语言

特征点 & 描述子keypoints & descriptors

在每张图上找出醒目的关键点，再用一串数字（SIFT 描述子）描述它周围长什么样。这是不同照片之间「对话」的语言。

◆ 建立连接

匹配 & 几何验证matching

比对描述子找出对应点，再用对极几何+RANSAC剔除错配，留下可信的「内点」。模式：穷举 / 序列 / 词汇树 / 空间。

◆ 镜头建模

相机模型camera models

描述镜头如何把三维投影到像素：焦距、主点、畸变。常见 SIMPLE_RADIAL（默认好用）、PINHOLE、OPENCV、鱼眼……选「最能描述你镜头的最简单模型」。

◆ 真理裁判

光束法平差bundle adjustment

整条流程的灵魂。用 Ceres 做非线性优化，同时微调所有相机参数和三维点，让它们重投影回照片的误差最小。

◆ 相机在哪

位姿：内参 & 外参pose

外参=相机在世界中的位置与朝向；内参=镜头自身的参数。SfM 的核心产物就是每张图的外参。

◆ 项目账本

数据库database

一个 SQLite 文件，存下相机、图像、关键点、描述子、匹配和双视图几何——整个工程的单一事实来源。

◆ 多相机

相机装置 & 帧rigs & frames

自 COLMAP 3.12.0（2025-06-30）起，可把多个相机刚性绑成一个 rig，同一时刻采集的一组测量叫一帧。适用于立体 / AR-VR / 车载场景。

◆ 三角化

从匹配到三维点triangulation

同一个点在多张已知位姿的照片里被看到，射线一交会，就还原出它的三维坐标——稀疏点云就是这么一点点长出来的。

◆ 稠密之眼

PatchMatch 立体MVS depth

MVS 的主力：为每个像素估计深度，得到深度图+法向图，再跨视图融合成稠密点云。可 GPU 加速。

04

三种 SfM 流程，怎么选？Three ways to do sparse reconstruction

默认 · 最稳

增量式

colmap mapper

一张一张地把图加进来，每加一张就三角化、平差。最稳健、测试最充分。

适合 大多数场景；缺点是图很多时偏慢。
最快 · 大规模

全局式 GLOMAP

colmap global_mapper

用旋转平均 + 全局定位一次性解算所有相机位姿，速度快得多。

适合 大而连通 的数据集；对外点更敏感，依赖好的焦距先验。
超大规模

分层式

hierarchical_mapper

把场景切成相互重叠的子模型，各自重建后再合并。

适合 增量式都嫌慢 的超大数据；稳健性通常略逊另两者。

上手只要几行Get started in minutes

不想碰参数？一键自动重建把整条流水线跑完。想要可视化？开图形界面。想写代码？用 PyCOLMAP。

终端 · 一键重建
# 打开图形界面
colmap gui

# 或：一条命令跑完稀疏+稠密
colmap automatic_reconstructor \
    --workspace_path ./project \
    --image_path     ./images
Python · PyCOLMAP
# pip install pycolmap
import pycolmap

pycolmap.extract_features(db, img_dir)
pycolmap.match_exhaustive(db)
maps = pycolmap.incremental_mapping(
    db, img_dir, out_dir)
maps[0].summary()

想拆开手动控制？依次是 feature_extractor → exhaustive_matcher → mapper → image_undistorter → patch_match_stereo → stereo_fusion。

06

5 条实战贴士Hard-won tips

01好照片 = 好模型。 纹理丰富、相邻图高度重叠、多视角环绕、光照稳定、对焦清晰。垃圾进，垃圾出。
02相机模型拿不准，就从 SIMPLE_RADIAL 开始，再看模型统计里的重投影误差决定要不要换更复杂的。参数过多反而会过拟合。
03按规模选流程： 默认增量式；又大又连通上全局式（GLOMAP）；超大规模才用分层式。
04开 GPU。 特征提取和 PatchMatch 稠密都能 GPU 加速，速度天差地别。
05稀疏没做好，稠密别强求。 MVS 完全依赖 SfM 的相机位姿——先确认稀疏模型干净、相机都注册上了。
07

黑话速查Decode the jargon

SfM
运动恢复结构，从多视角照片反推相机位姿与稀疏三维结构。
MVS
多视图立体，基于已知位姿重建稠密三维。
BA
光束法平差，联合优化相机与三维点、最小化重投影误差。
关键点/描述子
图像里醒目的点 / 描述其外观的数字向量。
内点/外点
几何验证后保留的正确匹配 / 被剔除的错误匹配。
对极几何
两张图之间的几何约束，用来验证匹配是否合理。
三角化
由多条视线交会求出三维点坐标。
位姿 / 内参 / 外参
相机的位置朝向 / 镜头自身参数 / 在世界中的姿态。
深度图
每个像素到相机的距离，MVS 的中间产物。
点云 / 网格
离散三维点的集合 / 连成面的三维表面。
